Beschreibung

Mit dem Application Load Balancer können sie die eingehenden Anfragen an eine Zielgruppe mit einer Lambda-Funktion als Ziel weiterleiten. Elastic Load Balancing verteilt Ihren eingehenden Datenverkehr automatisch auf mehrere Ziele, wie EC2-Instanzen, Container und IP-Adressen, in einer oder mehreren Availability Zones.

Es überwacht den Zustand seiner registrierten Ziele und leitet den Datenverkehr nur an die fehlerfreien Ziele weiter. Elastic Load Balancing skaliert Ihren Load Balancer, wenn sich Ihr eingehender Datenverkehr im Laufe der Zeit ändert. Es kann automatisch auf die überwiegende Mehrheit der Arbeitslasten skaliert werden.

Elastic Load Balancing unterstützt die folgenden Load Balancer: Application Load Balancer, Network Load Balancer, Gateway Load Balancer und Classic Load Balancer.

Sie können den Load-Balancer-Typ auswählen, der Ihren Anforderungen am besten entspricht. In diesem Leitfaden werden Anwendungs-Load-Balancer erläutert. Ein Load Balancer dient als zentraler Kontaktpunkt für Clients. Clients senden Anforderungen an den Load Balancer, die dieser wiederum an Ziele wie z. B. EC2-Instances sendet.

Zur Konfiguration Ihres Load Balancers erstellen Sie Zielgruppen und registrieren anschließend Ziele bei den Zielgruppen. Außerdem erzeugen Sie Listener für Verbindungsanforderungen von Clients und Listener-Regeln zum Weiterleiten von Anforderungen von Clients an Ziele in einer oder mehreren Zielgruppen.

Zielgruppen mit Lambda-Funktionen als Ziel: Application Load Balancer kann eingehende Anfragen an eine Zielgruppe mit einer Lambda-Funktion als Ziel weiterleiten.

Automatische Verteilung auf verschiedene Ziele: Elastic Load Balancing verteilt eingehenden Datenverkehr automatisch auf EC2-Instanzen, Container und IP-Adressen in einer oder mehreren Availability Zones.

Überwachung und Weiterleitung an fehlerfreie Ziele: Der Load Balancer überwacht den Zustand der registrierten Ziele und leitet den Datenverkehr nur an die funktionierenden Ziele weiter.

Skalierung entsprechend dem Datenverkehr: Elastic Load Balancing skaliert den Load Balancer automatisch, um sich änderndem eingehendem Datenverkehr gerecht zu werden.

Unterstützung verschiedener Load Balancer-Typen: Elastic Load Balancing unterstützt verschiedene Load Balancer-Typen, darunter Application Load Balancer, Network Load Balancer, Gateway Load Balancer und Classic Load Balancer.

Auswahl des geeigneten Load Balancer-Typs: Kunden können den Load-Balancer-Typ wählen, der ihren Anforderungen am besten entspricht.

Zentraler Kontaktpunkt für Clients: Ein Load Balancer fungiert als zentraler Kontaktpunkt für Clients, die Anfragen an ihn senden.

Konfiguration durch Erstellung von Zielgruppen und Registrierung von Zielen: Zur Konfiguration des Load Balancers erstellen Kunden Zielgruppen und registrieren Ziele bei den Zielgruppen.

Erstellung von Listenern und Listener-Regeln: Kunden erzeugen Listener für Verbindungsanforderungen von Clients und definieren Listener-Regeln zum Weiterleiten von Anfragen von Clients an Ziele in einer oder mehreren Zielgruppen.
