Beschreibung

AWS CodeCommit ist ein vollständig verwalteter Quellcode-Service, der sichere Git-basierte Repositorys hostet und Teams die Zusammenarbeit am Code in einer sicheren und hoch skalierbaren Umgebung ermöglicht. Der Dienst eliminiert die Notwendigkeit, ein eigenes Quellcodesystem zu betreiben oder sich Gedanken über die Skalierung der Infrastruktur zu machen. Code Commit unterstützt das sichere Speichern von Quellcode bis hin zu Binärdateien und integriert sich nahtlos mit Ihren vorhandenen Git-Tools.

Der Service macht das Hosten, Warten, Sichern und Skalieren Ihrer Quellcodeverwaltungsserver überflüssig, indem er automatisch den wachsenden Anforderungen Ihres Projekts gerecht wird. Die automatische Verschlüsselung Ihrer Dateien während der Übertragung und im Ruhezustand gewährleistet Sicherheit. AWS Code Commit ist in AWS Identity and Access Management (IAM) integriert, was es ermöglicht, den Benutzerzugriff auf Repositorys individuell anzupassen. Der Dienst unterstützt alle Git-Befehle und arbeitet reibungslos mit Ihren bestehenden Git-Tools zusammen.

Entwickler können ihre bevorzugten Plugins für Entwicklungsumgebungen, kontinuierliche Integrations- und Bereitstellungssysteme sowie grafische Clients nutzen. Das skalierbare Git-Repository von CodeCommit ist nahtlos in andere AWS-Dienste integriert und ermöglicht Entwicklern, Änderungen am Code zu verfolgen, verschiedene Versionen zu speichern und den Code mit Teamkollegen zu teilen. AWS Code Commit erleichtert Unternehmen die sichere und hoch skalierbare Hosting von privaten Git-Repositorys, ohne die Notwendigkeit, ein eigenes Quellcode-Verwaltungssystem zu implementieren oder sich Gedanken über die Skalierung der Infrastruktur zu machen.

Durch die Nutzung von Code Commit können Sie alles vom Quellcode bis zu Binärdateien sicher speichern und dabei die Geschwindigkeit und Häufigkeit Ihres Entwicklungslebenszyklus erhöhen. Der Service unterstützt Kollaboration durch Funktionen wie Pull-Anfragen, Verzweigungen und Zusammenführungen, während Sie die Kontrolle darüber behalten, wer Änderungen an bestimmten Zweigen vornehmen darf. Ihre Repositorys bleiben in der Nähe Ihrer Build-, Staging- und Produktionsumgebungen in der AWS-Cloud, wodurch Sie auch inkrementelle Änderungen effizient implementieren können.
