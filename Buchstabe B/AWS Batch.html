Beschreibung

AWS Batch ermöglicht die effiziente Planung und Ausführung von Batch-Computing-Workloads über verschiedene AWS-Computing-Services. Der Service automatisiert die Bereitstellung optimaler Rechenressourcen, wie CPU- oder speicheroptimierte Instanzen, basierend auf dem Volumen und den spezifischen Anforderungen der übermittelten Batch-Jobs. Dabei optimiert AWS Batch die Verteilung von Rechenressourcen je nach Volumen und Anforderungen der übermittelten Batch-Jobs. Mit AWS Batch können Entwickler, Wissenschaftler und Ingenieure Batch-Verarbeitungs-Workloads in der AWS Cloud ausführen, ohne sich um die Konfiguration und Verwaltung der Infrastruktur kümmern zu müssen. Der Service befreit von undifferenziertem Aufwand und ermöglicht die effiziente Bereitstellung von Ressourcen als Reaktion auf gesendete Aufträge. Dies hebt Kapazitätsbeschränkungen auf, reduziert IT-Kosten und beschleunigt die Bereitstellung von Ergebnissen.

Komponenten von AWS Batch:

Aufträge:

-Eine Arbeitseinheit, z. B. ein Shell-Script oder ein Docker-Containerabbild, die an AWS Batch gesendet wird.
-Wird als containerisierte Anwendung auf AWS Fargate oder Amazon EC2-Ressourcen ausgeführt.
-Kann auf andere Aufträge verweisen und vom erfolgreichen Abschluss anderer Aufträge abhängen.

Auftragsdefinitionen:

-Legt fest, wie Jobs ausgeführt werden sollen und dient als Blaupause für Ressourcen.
-Weist dem Job eine IAM-Rolle zu, um Zugriff auf andere AWS-Ressourcen zu gewähren.
-Spezifiziert Speicher- und CPU-Anforderungen sowie Containereigenschaften und Umgebungsvariablen.

Auftragswarteschlangen:

-Jobs werden an eine bestimmte Auftragswarteschlange gesendet, bis sie in einer Rechenumgebung eingeplant werden.
-Verknüpft mit einer oder mehreren Datenverarbeitungsumgebungen, denen Prioritätswerte zugewiesen werden können.

Datenverarbeitungsumgebung:

-Gruppe verwalteter oder nicht verwalteter Datenverarbeitungsressourcen für die Ausführung von Aufträgen.
-In verwalteten Umgebungen kann der gewünschte Rechentyp und weitere Details spezifiziert werden.
AWS Batch startet, verwaltet und beendet Computetypen effizient nach Bedarf.

Als vollständig verwalteter Service eliminiert AWS Batch die Notwendigkeit, Batch-Computing-Software zu installieren oder zu verwalten. Dies ermöglicht es den Nutzern, sich auf die Analyse von Ergebnissen und die Lösung von Problemen zu konzentrieren.

AWS Batch automatisiert die Planung und Ausführung von Batch-Computing-Workloads in der AWS Cloud.
*Entwickler können Batch-Jobs ohne Infrastrukturverwaltung ausführen.

*AWS Batch verwaltet effizient Rechenressourcen basierend auf Bedarf.

*Es ist ein vollständig verwalteter Service, der die Installation und Verwaltung von Batch-Computing-Software überflüssig macht.
